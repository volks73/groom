The error type for groom-related operations and associated traits.

Errors mostly originate from the dependencies, but custom instances of Error can be created with the Generic variant and a message. If the error is related to user input, then the Input variant should be used.

Variants

Generic(String)

A generic or custom error occurred. The message should contain the detailed information.

Input(String)

An error occurred with end-user input, such as a typo in a path.

Io(Error)

An I/O operation failed.

Mustache(Error)

Parsing and rendering a mustache template failed.

Utf8(Utf8Error)

Converting to and/or from a UTF8 string failed.

Yaml(Error)

Decoding or deserializing YAML data failed.

Gets an error code related to the error.

This is useful as a return, or exit, code for a command line application, where a non-zero integer indicates a failure in the application. It can also be used for quickly and easily testing equality between two errors.
